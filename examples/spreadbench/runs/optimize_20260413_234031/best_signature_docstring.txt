Manipulate a spreadsheet according to the given instruction.

    1. Load the input spreadsheet from /sandbox/input/input_spreadsheet/ using openpyxl.
    2. Inspect thoroughly before writing any code:
       a. List all sheet names, row counts, and column headers.
       b. Print sample data rows (first 5–10 AND last 5) to understand the full data range.
       c. If the instruction references specific cells/ranges, print their current values.
       d. If any cells already contain example answers, study them carefully to infer the expected pattern (e.g., per-group vs global calculations, exact string formatting, column-to-column relationships).
       e. Identify the EXACT target cells/columns where output should be written. Cross-reference the instruction with existing column headers, existing formulas, and any blank columns that appear designated for output. Do NOT assume — verify by printing the relevant area.
       f. If a target cell/column already contains a formula, print it. Understanding the existing formula is critical — it tells you what the user currently has and what they want changed.
    3. Perform the actual data manipulation in Python using openpyxl — do NOT just write VBA/macro code as text into cells. Even if the user phrases the task as "write a VBA macro to …" or "create a VBA code to …", you must execute the equivalent operation directly on the spreadsheet data.
    4. Apply operations to ALL relevant rows, not just a subset. Use ws.max_row (or scan for the last non-empty row) to determine the true data extent. Never hard-code row limits or assume a small number of rows. Conversely, do NOT write values to rows that should remain empty — check which rows actually need output.
    5. Preserve the original sheet names and structure. NEVER create new sheets, delete existing sheets, or rename sheets unless the instruction explicitly asks for it. Write results into the EXISTING sheet(s). If the instruction says "display results in column B and column C", write into columns B and C of the SAME sheet — do not create a new sheet named "output" or similar.
    6. When the instruction asks for a formula (e.g., COUNTIF, SUMIF, SUMPRODUCT, VLOOKUP, etc.), write an actual Excel/LibreOffice formula string into the cell — do NOT compute the result in Python and write a static value. Formulas ensure the spreadsheet remains dynamic. Only use Python-computed static values when the instruction clearly asks for a one-time computation or when no formula can express the logic.
    7. CRITICAL — When modifying an existing formula (e.g., wrapping it in IF()), strip the leading "=" from the original formula before embedding it. For example, if the original cell value is '=IF(C9="","",...)', the new formula should be '=IF(E9=F9,"PAID",IF(C9="",...))' — NOT '=IF(E9=F9,"PAID",=IF(C9="",...))'. A nested "=" causes #VALUE! errors in Excel/LibreOffice.
    8. When the user describes an expected result (e.g., "the answer should be 528"), treat this as a hint, NOT as the ground truth. Always compute or construct the formula based on the actual data. The user's stated value may be approximate, rounded, or wrong.
    9. Do NOT give up if data appears missing at first glance. Look more carefully: check all sheets, check whether row 1 vs row 2 is the header, check for merged cells, check if the data layout differs from your assumption. Print more rows and columns before concluding data is absent.
    10. For conditional formatting tasks: in addition to (or instead of) adding openpyxl CF rules, compute the condition in Python and directly set cell fills using PatternFill. CF rules alone may not produce values that survive recalculation scoring. Apply the fill directly: cell.fill = PatternFill(start_color="FFFF00", end_color="FFFF00", fill_type="solid").
    11. Before modifying cell values, check for pre-existing data that already satisfies the condition. For example, if the task says "append '-Fuel' to column I", first check if the cell already ends with '-Fuel' to avoid double-appending. Read and print the current values before deciding what to change.
    12. Save to /sandbox/output/output_spreadsheet/ with the SAME filename as the input.
    13. After saving, call recalculate() on the output file to evaluate any written formulas.
    14. IMPORTANT: recalculate() may change sheet names (e.g., "Sheet1" → "Ark1" due to LibreOffice locale). After recalculate, reload the file and check that sheet names still match the original. If a sheet was renamed, rename it back to the original name, save again, and do NOT call recalculate a second time.
    15. Verify the output by reloading the saved file (data_only=True after recalculation) and printing every target cell/range mentioned in the instruction. Confirm:
       a. No target cells are None or blank when they should have values.
       b. If any target cells are None after recalculation, the formula may be incompatible with LibreOffice. Debug: print the formula string, check for syntax issues (especially nested "=" signs, mismatched parentheses, or Google-Sheets-only functions), try alternative formula syntax, re-save, re-recalculate, and re-verify.
       c. String values have no unintended leading/trailing whitespace.
       d. Numeric values are plausible (sanity-check against the input data).
       e. All sheets expected in the output actually exist with correct names.
       f. The number of output rows/columns matches what the instruction requires.
       g. Verify the EXACT target column — if the instruction says column H, confirm you wrote to column H (column index 8), not column F or another column. Count columns explicitly using openpyxl.utils.column_index_from_string().
       h. If formulas evaluate to #N/A, #VALUE!, or #REF!, the formula is wrong. Do NOT submit — debug the formula syntax, fix it, re-save, re-recalculate, and re-verify.
    16. If verification reveals mismatches, debug and fix before submitting. Re-save, re-recalculate, and re-verify.
    17. When the task involves extracting or filtering data, double-check the filtering condition against the actual data. Print the rows that match and those that don't to confirm the filter is correct. If the instruction says "values corresponding to rows where there's an 'X'", verify which rows have 'X' and in which column.
    18. When the task involves complex formula logic (VLOOKUP with conditions, INDEX/MATCH with multiple criteria), consider implementing the logic in Python instead of as a formula if the formula approach fails verification. Write the computed values directly to cells. This is acceptable when formula syntax is too complex or incompatible with LibreOffice.
